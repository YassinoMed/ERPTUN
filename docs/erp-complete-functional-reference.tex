\section{Objet du document}
Ce document decrit l'ERP Laravel modulaire present dans le projet \texttt{main-file}. Il centralise les axes fonctionnels, les modules, les fonctionnalites, les briques transverses, les verticales metier, les composants SaaS et les services techniques deja integres dans l'application.

Le document couvre:
\begin{itemize}[leftmargin=1.2cm]
    \item l'architecture generale de l'ERP;
    \item les domaines fonctionnels transverses;
    \item les modules metier par verticale;
    \item les fonctions SaaS, securite, API, workflows et UX;
    \item l'etat de couverture fonctionnelle de la plateforme.
\end{itemize}

\section{Vue d'ensemble de la plateforme}
\subsection{Nature du produit}
L'application est un ERP SaaS multi-tenant construit sur Laravel et organise autour:
\begin{itemize}[leftmargin=1.2cm]
    \item d'un coeur applicatif dans \texttt{app/};
    \item de modules additionnels dans \texttt{Modules/};
    \item d'une couche Blade pour l'interface;
    \item d'un modele de permissions, plans SaaS et activation par modules;
    \item de services transverses pour les settings, logs, numerotation, audit, workflow, automation, securite et API.
\end{itemize}

\subsection{Axes couverts}
\begin{itemize}[leftmargin=1.2cm]
    \item ERP generaliste: CRM, ventes, achats, stock, finance, RH, projets.
    \item Operations: production, maintenance, qualite, WMS, POS, livraison.
    \item Verticales: industrie, agroalimentaire, medical, immobilier, assurance.
    \item Corporate: gouvernance, filiales, innovation, documentation, conformite.
    \item SaaS: plans, addons, onboarding tenant, quotas, portail, API.
\end{itemize}
